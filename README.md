## 🔔 Как установить напоминания о ритуалах

Методология MSW основана на регулярных ритуалах (ежедневные заметки, воскресная ретроспектива). Вот как настроить автоматические напоминания:

### Для пользователей Cursor (автоматически)

Если вы используете **Cursor IDE**, AI автоматически напомнит о ритуалах:
- Воскресенье: "День ретроспективы!"
- Понедельник: "Время планирования недели!"
- Пятница: "Завершайте рабочие продукты!"

**Ничего настраивать не нужно** — напоминания встроены в `.cursorrules`.

### Для всех остальных (вручную)

Выберите удобный способ:

#### Вариант 1: Google Calendar (рекомендуется)

**Шаг 1: Создать recurring event**
1. Откройте Google Calendar
2. Создайте новое событие "Воскресная ретроспектива MSW"
3. Настройки:
   - **Время:** Каждое воскресенье, 19:00-19:30 (или удобное вам)
   - **Повторение:** Каждую неделю
   - **Уведомление:** За 30 минут
   - **Описание:** "30 мин: Ретро + Разбор заметок + Планирование"

**Шаг 2: Добавить ежедневное напоминание (опционально)**
- Событие "Ежедневные заметки MSW"
- Время: каждый день, например 21:00-21:15
- Повторение: Ежедневно

**Шаблоны для Google Calendar по неделям:**
- Неделя 1: Ежедневные заметки (15 мин)
- Неделя 2: + Воскресная ретроспектива (30 мин)
- Неделя 4: + Понедельник планирование (30 мин)

#### Вариант 2: Телефон (будильник/напоминание)

**iOS (Напоминания):**
1. Создать напоминание "Ретроспектива MSW"
2. Повторение: Каждое воскресенье, 19:00
3. Уведомление: Включить

**Android:**
1. Google Keep / Samsung Reminders
2. Recurring notification каждое воскресенье

#### Вариант 3: Notion (для пользователей Notion)

**Recurring task в Notion:**
1. Создать database "MSW Rituals"
2. Добавить задачу "Воскресная ретроспектива"
3. Properties:
   - Date: Recurring every Sunday
   - Template: Weekly_Retro_Simple.md
4. Настроить Notion Reminders

#### Вариант 4: Obsidian (для пользователей Obsidian)

**Плагин "Reminder":**
1. Установить плагин "Reminder" или "Tasks"
2. Создать note "MSW Weekly Rituals"
3. Добавить recurring tasks:
   ```markdown
   - [ ] Воскресная ретроспектива @repeat(every Sunday at 19:00)
   - [ ] Ежедневные заметки @repeat(every day at 21:00)
   ```

### Какие напоминания нужны по неделям

| Неделя | Ежедневно | Воскресенье | Понедельник |
|--------|-----------|-------------|-------------|
| **1** | Заметки 15 мин | — | — |
| **2-3** | Заметки 15 мин | Ретроспектива 30 мин | — |
| **4+** | Заметки 15 мин | Сессия стратегирования 60 мин | Планирование 30 мин |

### Шаблоны для использования

- **Неделя 2-3:** `docs/facilitators/templates/Weekly_Retro_Simple.md` (упрощённая, 30 мин)
- **Неделя 4+:** `docs/facilitators/templates/Weekly_Retro_Template.md` (полная, 60 мин)
- **Планирование:** `templates/Week_04_Weekly_Sprint_Template.md`
- **Чек-лист сессии:** `checklists/Checklist_Weekly_Strategizing.md`

### Важно

**Регулярность важнее инструмента:**
- Выберите ОДИН способ напоминаний и придерживайтесь его
- Лучше простой будильник на телефоне каждую неделю, чем сложная система, которую не используете

**Первые 3-4 недели критичны:**
- Формируется привычка к ритуалам
- Пропустите 2 недели подряд → ритуал разрушается
- Держитесь первые 4 недели → ритуал становится автоматическим

### Course CLI — Работа через командную строку

**Быстрый старт:**
```bash
# Установка
cd course_cli
pip install -e .
cd ..

# Основные команды
course start-week 1          # Начать неделю 1
course contract init         # Создать личный контракт
course progress              # Показать прогресс
course template --list       # Список шаблонов
course info                  # Информация о курсе
```

**Что даёт:**
- ✅ Быстрая навигация по неделям через чат в Cursor/IDE

- ✅ Удобная работа с шаблонами

### 🎯 Настройка Cursor для автоматической работы

**Для полной интеграции с Cursor:**
1. Установите CLI (см. выше)
2. Откройте проект в Cursor
3. Cursor автоматически подхватит настройки из `.cursorrules`

**Теперь можете работать через чат:**
- "Начни неделю 1" → автоматически откроет материалы

### 🔧 Для разработчиков и поддержки проекта

**Система автоматического контроля качества**

Мы настроили систему, которая автоматически проверяет, что вся документация курса актуальна и корректна. Это означает, что участники всегда получают самую свежую и точную информацию.

**Как это работает:**
- 📋 **Автоматическая проверка** — система сама следит за обновлениями
- 🔍 **Контроль качества** — проверяет, что все ссылки работают
- ⚡ **Мгновенные обновления** — изменения сразу отражаются в документации

**Технические детали:** см. `scripts/README.md`

## Три способа работы

### 1. Традиционный (как раньше)

- Редактируете контракты в любом редакторе

### 2. Через CLI (командная строка)
- Работаете через команды `course`
- Быстрая навигация и создание файлов
- Отслеживание прогресса

### 3. Через Cursor (с AI-ассистентом)

- Cursor автоматически выполняет CLI команды
- Полная интеграция с контекстом курса
